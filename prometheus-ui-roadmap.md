# Prometheus UI 개선 로드맵

## 진행 방식

각 항목은 아래 순서로 진행한다.

1. 구현 범위를 작게 잡는다.
2. 해당 항목만 수정한다.
3. `npm run tsc:check`로 타입 검증을 한다.
4. 실제 UI 흐름을 확인한다.
5. 문제가 없으면 체크한다.
6. 다음 항목으로 넘어간다.

한 단계에서 문제가 생기면 다음 단계로 넘어가지 않고, 해당 단계 안에서 원인을 확인한 뒤 수정한다.

## 체크리스트

- [x] tool call을 큰 카드가 아니라 한 줄 timeline 형태로 표시한다.
- [x] 실행 전, 실행 중, 완료, 실패 상태별 문구를 정리한다.
- [x] 긴 명령어와 파일 경로는 한 줄 말줄임 처리한다.
- [x] 클릭하면 기존 상세 내용을 펼칠 수 있게 유지한다.
- [x] 기본 상태는 접힘으로 둔다.
- [x] 타입 체크를 통과한다.
- [x] 실제 실행 UI를 확인한다.
- [x] 검토 완료 후 이 항목을 체크한다.

### 2. 파일 생성/수정 tool 전용 상태 문구 정리

- [x] 파일 생성 tool과 수정 tool을 구분하는 기준을 확인한다.
- [x] 생성 예정 상태 문구를 정리한다.
- [x] 생성 중 상태 문구를 정리한다.
- [x] 파일 생성 완료 상태 문구를 정리한다.
- [x] 파일 수정 완료 상태 문구를 정리한다.
- [x] 실패 상태 문구와 아이콘을 정리한다.
- [x] 완료 상태는 녹색 체크 아이콘으로 표시한다.
- [x] 타입 체크를 통과한다.
- [x] 실제 파일 생성/수정 UI를 확인한다.
- [x] 검토 완료 후 이 항목을 체크한다.

### 3. Tool 상세 기본 접힘 유지 및 diff 스타일 개선

- [x] 현재 파일 결과 표시 컴포넌트를 확인한다.
- [x] 파일 상세는 기본 접힘 상태로 유지한다.
- [x] 펼쳤을 때 파일명, 추가/삭제 라인 수를 표시한다.
- [x] 신규 파일은 초록 추가 라인 중심의 diff 스타일로 표시한다.
- [x] 긴 파일은 UI가 깨지지 않도록 높이 제한 또는 접힘 처리를 적용한다.
- [x] 복사/열기 버튼 위치를 정리한다.
- [x] 타입 체크를 통과한다.
- [ ] 실제 diff 표시를 확인한다.
- [ ] 검토 완료 후 이 항목을 체크한다.

### 4. 여러 tool call을 그룹으로 묶기

- [x] 한 assistant 응답 안의 tool call 묶음 구조를 확인한다.
- [x] 명령어 실행 묶음을 `명령어 N개 실행` 형태로 요약한다.
- [x] 파일 생성 묶음을 `파일 N개 생성` 형태로 요약한다.
- [x] 파일 수정 묶음을 `파일 N개 수정` 형태로 요약한다.
- [x] 그룹은 기본 접힘 상태로 둔다.
- [x] 펼치면 개별 timeline 항목을 보여준다.
- [x] 타입 체크를 통과한다.
- [ ] 여러 tool call이 섞인 응답을 확인한다.
- [ ] 검토 완료 후 이 항목을 체크한다.

### 5. 최종 작업 요약 섹션 추가

- [ ] 작업 시작/종료 시각 또는 duration을 계산할 수 있는 위치를 확인한다.

- [ ] 주요 생성 파일 목록을 표시한다.
- [ ] 검증 명령 결과를 표시할 수 있는 구조를 확인한다.
- [ ] 실행 방법 안내를 최종 assistant 응답 아래에 정리한다.
- [ ] README, 요구사항 문서 같은 주요 파일을 카드 형태로 표시한다.
- [ ] 타입 체크를 통과한다.
- [ ] 전체 작업 완료 후 요약 표시를 확인한다.
- [ ] 검토 완료 후 이 항목을 체크한다.

### 6. 최종 변경 파일 review 패널 추가

- [ ] 변경 파일 목록을 가져오는 기존 로직이 있는지 확인한다.
- [ ] 전체 변경 파일 수와 총 추가/삭제 라인 수를 표시한다.
- [ ] 파일별 추가/삭제 라인 수를 표시한다.
- [ ] 각 파일은 접힘/펼침 가능하게 만든다.
- [ ] review 버튼 위치를 만든다.
- [ ] 실행 취소 버튼은 UI 자리만 먼저 만들고 실제 기능은 별도 단계로 분리한다.
- [ ] 타입 체크를 통과한다.
- [ ] 변경 파일이 많은 작업에서 UI를 확인한다.
- [ ] 검토 완료 후 이 항목을 체크한다.

## 우선순위

먼저 1번과 2번을 진행한다.

이 두 항목이 체감 UI의 핵심이다. 여기서 안정적으로 보이면 3번부터 6번까지 순서대로 진행한다.

## 각 단계 완료 기준

- [ ] 타입 체크가 통과한다.
- [ ] 기존 tool 실행 흐름이 깨지지 않는다.
- [ ] 자동 승인 tool과 수동 승인 tool이 모두 정상 동작한다.
- [ ] 기본 접힘 상태가 유지된다.
- [ ] 긴 명령어와 긴 파일 경로가 UI를 밀어내지 않는다.
- [ ] 사용자가 보기에는 작업 흐름이 timeline처럼 읽힌다.
