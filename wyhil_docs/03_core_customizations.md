# 3. 플러그인 주요 커스텀 사항 요약

오리지널 Continue 플러그인에서 사내 요구사항에 맞춰 크게 **어떤 부분을 얼마나 커스텀했는지**를 요약합니다. 이 수정 내역을 알아야 추후 Continue 오픈소스가 업데이트 되어 Merge를 할 때 충돌을 원활히 대처할 수 있습니다.

## 3.1 라우팅 및 LLM 연동 변경 (`core/`)

- 기존에 플러그인이 클라이언트 단에서 각각의 LLM 엔드포인트(OpenAI 서버, Anthropic 서버 등)를 바라보던 것을 차단했습니다.
- 대신 모든 Chat/Completion 통신이 `ai_proxy` (사내 처리 서버)의 특정 URI(예: `/chat/completions`)를 거치도록 하드코딩 또는 설정 오버라이딩을 적용했습니다.

## 3.2 Security Scan (보안 검사 및 자동차단/수정 기능)

사내 보안 강화의 일환으로 아래 두 기능을 신규 개발 및 연동했습니다.

- **수동 스캔 (IDE 메뉴 / Slash 커맨드 등):** 보안 결함이 있는지 코드를 서버(`/scan`)로 보내 검사하고 결과를 마크다운 UI로 전시.
- **파일 저장 시(On-Save) 자동 스캔:** `vscode` 및 `intellij` 확장 브릿지에서 모니터링하다가 파일이 저장될 때 백그라운드로 스캔을 띄워, 만약 위험 코드가 감지되면 경고창을 표시합니다 (IntelliJ Security Loop 방지 등 이슈 해결 포함).
- **Auto Fix (`/scan/fix`):** 발견된 보안 이슈를 LLM의 Tool Call(또는 Edit Tool) 기능을 활용해 IDE 상에 직접 파일 수정 제안(Diff View)으로 덮어씌웁니다.

## 3.3 사용자 인증 및 커스텀 Login UI (`gui/`)

- IDE 우측 메뉴 등에 사내 미니 대시보드(Login Dialog)를 구현하여 로그인 상태 관리, 헬스 체크 상태 등을 전시합니다.
- UI 디자인 개선: 사내 가이드를 따라 팝업 색상 보정, 불필요한 에러 텍스트 제거, 로그인 상태값 및 서버 URL 표시 방식 등을 대대적으로 리디자인 했습니다 (예: Unknown Status -> 확인전 등).

## 3.4 Context Manager 개편

대규모 사내 코드를 담기 위해 오래된 대화나 불필요한 컨텍스트 토큰을 단순히 자르는 방식에서, **압축 메커니즘을 수정하여 지정된 길이를 초과 시 가장 오래된 Q&A 쌍을 제거**하는 식으로 로직을 단순화/최적화했습니다 (`context_manager` 로직 개편).

## 3.5 특정 모델(gpt-4o, gemini 등) 툴 지원 보정

`gpt-4o`, `gemini` 등 사용하는 처리 서버 뒷단의 LLM 모델 특성에 맞춰 Tool Call 정의가 인식이 안 되는 문제들을 회피하기 위해 `core/llm/toolSupport.ts` 등을 수정해서 모델별 Tool Parsing 파이프라인을 커스텀했습니다.
